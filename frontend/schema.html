Core Database Schema

A rich dataset can be constructed around virus nucleotide data. The data within a V-gDB project is structured according to the Core Schema; a fixed set of object types and relationships.

The V-gDB database is generated using the Viral Genome Toolkit.

Meta_data

The meta_data table stores descriptive information about each sequence, identified by the NCBI primary_accession number. It contains sequence information such as length, clade assignment, strain. It also contains sample details such as isolate_id, host species, and country of origin.

Sequences

The sequences table contains the NCBI primary_accession number and the associated unaligned sequece.

Features

The features table contains the coordinates of each feature in the aligned sequence with regards to its aligned reference sequence.

  • The cds_start and cds_end are the coordinates of the feature.
  • The aln_start and aln_end are the coordinates of the entire aligned sequence.

Sequence_alignment

The sequence_alignment table contains the NCBI primary accession of the sequence and its reference sequence and the aligned sequence. The aligned sequence does not include the insertions. See the insertions table for this information.

Insertions

The insertions table contains the NCBI primary_accession of the sequence and reference and a comma seperated list of the starting nucleotide position and the insertions at that spot.
